## 1. ТЕХНИЧЕСКОЕ SEO

### 1.1 Robots.txt

**Файл:** https://switchonai.ru/robots.txt
**Статус:** ⚠️ Проблемный

```
ПРОБЛЕМА: Disallow: *page
```
Это правило блокирует ВСЕ URL, содержащие подстроку "page":
- `/sample-page/` — заблокирована
- Пагинация `/page/2/` — заблокирована
- Любые URL с "page" в пути

Позитивно:
- Правильно заблокированы `/wp-admin/`, `/cgi-bin/`, `/xmlrpc.php`
- Заблокированы `/author/`, `/search/`, `/feed/`
- UTM-параметры в `Clean-param` для Яндекса — ✅

**Проблема: Несоответствие sitemap ↔ robots.txt**
- `author-sitemap.xml` существует и индексируется
- Но `/author/` заблокирован в robots.txt — URL попадают в sitemap, но не индексируются

### 1.2 Sitemap

**Файл:** https://switchonai.ru/sitemap_index.xml
**Статус:** ⚠️ Есть проблемы

- WordPress + множество плагинов — типичный риск раздутых ресурсов

### 1.5 Аналитика

- **Google Analytics**: MonsterInsights установлен, НЕ НАСТРОЕН. Комментарий прямо в коде: _"MonsterInsights в настоящее время не настроен на этом сайте"_. Данные не собираются с момента запуска сайта. Это критично.

## 3. КОНТЕНТ И E-E-A-T

### 3.1 Объём и структура

- **~170+ статей** в блоге (и продолжают публиковаться — последняя дата 2026-03-28)
- **1 категория "Рубрика"** для всех статей — нет таксономии вообще
- **Смешанные URL**: большинство на русском транслите, часть на английском (inconsistency)

```
Русские: /vnedrenie-ai-biznes-avtomatizaciya-rutiny/
Английские: /scalable-content-system-constructor-principle/
```

### 3.2 Каннибализация ключевых слов

Явная каннибализация:
- `/oshibki-vnedreniya-ai-v-biznes/` + `/oshibki-vnedreniya-ai-v-biznes-2/` + `/oshibki-vnedreniya-ii-v-biznese/` — 3 статьи на одну тему
- `/organicheskoe-prodvizhenie-video-youtube/` + `/...youtube-2/` + `/...youtube-3/` — 3 дубля
- `/operator-ai-v-biznese/` + `/operator-ai-v-biznese-2/` — дубль
- `/avtomatizaciya-rutinnyh-zadach-s-pomoshchyu-ii/` + `/avtomatizatsiya-rutinnyh-zadach-s-pomoschyu-ii/` — опечатка в slug, два URL на одну тему

### 3.3 Качество контента (E-E-A-T)

**Позитивно:**
- Автор чётко идентифицирован: Павел Лещенко, ИП, адрес, ИНН
- Реальная экспертность демонстрируется (собственный опыт внедрения)
- Хорошая структура статей (H2, H3), проверенная статья ~2000 слов

**Негативно:**
- Нет страницы "Об авторе" (author page заблокирована)
- Нет кейсов с результатами (только декларативные описания)
- Нет внешних ссылок / упоминаний / отзывов с верификацией
- Нет даты публикации, видимой читателю

### 3.4 Thin Content

Риск тонкого контента:
- `/stati/` — пустая страница без meta/H1
- `/price/` — страница с минимальным контентом, без мета
