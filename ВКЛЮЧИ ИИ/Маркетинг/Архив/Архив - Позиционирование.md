ЭТАП 7 — МАСШТАБИРОВАНИЕ (как расширять систему, не разваливая фокус)
Цель: после 30–60 дней работы по циклу C (контента нет/конкуренты давят) масштабировать:
- сценарии (JTBD),
- каналы,
- продукты,
без превращения стратегии в хаос.

=====================================================================
1) КОГДА МОЖНО МАСШТАБИРОВАТЬ (условия)
Масштабирование включаем только если выполнены 4 условия:

Условие 1 — Конвертер стабилен
- есть стабильные “разборы” (запросы)
- люди оставляют контакт
- есть созвоны

Условие 2 — TG база стала причиной остаться
- люди сохраняют/реагируют на промпты/подборки/обзоры
- есть повторные заходы/обратная связь

Условие 3 — Производство не ломает тебя
- YouTube Long выходит стабильно
- short’ы и статьи делаются без выгорания

Условие 4 — Есть хотя бы 1–2 мини-результата
- свой суб-кейс (R&D) или первые пилоты
- пусть даже маленькие, но в цифрах/фактах

Если хотя бы 2 пункта не выполнены — масштабировать рано.

=====================================================================
2) ЧТО МАСШТАБИРОВАТЬ В ПЕРВУЮ ОЧЕРЕДЬ (приоритеты)
Порядок расширения (самый безопасный):

3) Масштабируем доказательства
- больше разборов, больше примеров, больше “до/после”
- это повышает конверсию без роста охвата

2) Масштабируем дистрибуцию якоря
- больше перепаковок, больше статей под подзапросы (SEO/GEO)
- но якорь остаётся 1 YouTube Long в неделю

3) Масштабируем сценарии (JTBD) — только потом
- добавляем второй сценарий, когда первый уже приносит “разборы” стабильно

4) Масштабируем каналы
- второй основной канал охвата добавляем только если первый стабилен

=====================================================================
3) КАК ДОБАВЛЯТЬ ВТОРОЙ JTBD-СЦЕНАРИЙ (чтобы не смешать всё)
После цикла C добавляем один сценарий на выбор:

Вариант 1 (логичное продолжение): G “трафик есть, денег мало”
- для тех, у кого контент уже появился/есть вход, но не дожимает до покупки

Вариант 2 (ускорение): B “контент есть, хочу ускорить производство”
- под обзоры/сборки/инструменты, сильный формат для YouTube

Правило внедрения второго сценария:
- не меняем весь контент
- добавляем 20–30% материалов под новый сценарий
- остальное остаётся под основной C

Пример распределения:
- 70% C
- 30% (G или B)

=====================================================================
4) КАК ДОБАВЛЯТЬ ВТОРОЙ ОСНОВНОЙ КАНАЛ ОХВАТА
Добавлять второй канал можно, если:
- первый канал даёт стабильные переходы в TG/разбор

Выбор второго канала по логике:
- если первый TikTok → вторым Shorts (или наоборот)
- Instagram добавляется, если ты готов вести сторис/закрепы и быстро отвечать

Правило:
второй канал получает перепаковку, а не отдельную стратегию.
Никаких “двух разных контент-планов”.

=====================================================================
5) КАК РАСШИРЯТЬ TG-БАЗУ (чтобы она становилась “продуктом”)
Расширяем базу по принципу:
- каждую неделю +2 объекта (промпт/подборка/обзор)
- раз в 2 недели +1 обзор контент-завода
- раз в месяц +1 “супергайд” (методичка в Telegraph + ссылка в TG)

Важно:
база должна быть навигируемой (карта) и обновляемой.

=====================================================================
6) КОГДА ДОБАВЛЯТЬ ПРОДУКТОВУЮ ЛИНЕЙКУ (клуб/платформа)
Пока нет устойчивого потока разборов и 1–2 пилотов:
- не выводим на витрину несколько продуктов

Когда появятся пилоты/результаты:
1) Основной вход остаётся “Разбор”
2) Дальше 2 направления:
- пилот/внедрение “под ключ”
- клуб/самостоятельное внедрение (если спрос есть)

Правило:
никогда не меняем “входную дверь”.
Меняем только то, что происходит после неё.

=====================================================================
7) РИСКИ МАСШТАБИРОВАНИЯ (что ломает систему)
- слишком рано добавить второй сценарий → контент расползётся
- слишком рано добавить второй канал → выгорание и падение качества
- слишком рано добавить 3–4 продукта → путаница и падение конверсии
- статьи “10/день” без качества и связки с TG → репутация “спам” и ноль разборов

=====================================================================
8) ПРАВИЛО МАСШТАБИРОВАНИЯ “1 ПЕРЕМЕННАЯ ЗА РАЗ”
В каждый новый 30–60 дневный цикл меняем только одно:
- либо сценарий (JTBD),
- либо канал,
- либо продуктовый шаг (после разбора),
- либо формат доказательств.

Если менять 2–3 переменные одновременно — невозможно понять, что сработало.

=====================================================================
9) ЧЕК-ЛИСТ РЕШЕНИЯ “ГОТОВЫ МАСШТАБИРОВАТЬ?”
Ответь “да/нет”:

- Есть стабильные переходы в TG?
- Есть стабильные “разборы”?
- Есть стабильные контакты через бот?
- Есть созвоны каждую неделю?
- Есть минимум 1–2 результата (хоть маленьких)?
- Производство якоря (YouTube) стабильно?
- TG база растёт и люди сохраняют?

Если “да” ≥ 5 пунктов — масштабируем.
Если меньше — сначала чиним узкое место.

=====================================================================
10) РЕЗУЛЬТАТ ЭТАПА 7
У тебя появляется:
- понятный режим расширения без хаоса
- правило “одна переменная за цикл”
- порядок: доказательства → дистрибуция → сценарии → каналы → линейка
