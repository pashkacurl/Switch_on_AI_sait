# Архитектура автоматизации Threads + Twitter + LinkedIn

## Полная система: 5 потоков контента на автопилоте

---

## ОБЩАЯ СХЕМА

Все 5 потоков генерируют тексты → складывают в единую Airtable → из Airtable автоматически публикуются на Threads + Twitter + LinkedIn по расписанию.

Один пост = один ряд в таблице с полями:

- text (основной пост)
- thread_2 (второй пост в ветке, если есть)
- source_type (из какого потока)
- platform (threads / twitter / linkedin / all)
- status (draft / ready / published)
- scheduled_time

---

## ПОТОК 1: "ИЗ СТАТЬИ" (автоматический, 4-5 постов/день)

**Источник:** Каждая статья которую публикует основной контент-завод.

**Триггер:** Появление новой статьи в Airtable.

**Агент:** Берёт тему + draft_snippet + заголовок → генерирует Thread-пост по вирусному формату.

**Промпт:** Уже готов (agent_threads_prompt.md). Ротация 6 форматов.

**Частота:** Автоматически с каждой статьёй = 4-5 постов/день.

**Мостик:** Ссылка на Telegram-канал.

---

## ПОТОК 2: "ИЗ TELEGRAM" (полуавтоматический)

**Источник:** Твои посты в Telegram-канале.

**Триггер:** Кнопка в Airtable или Telegram-бот: ты нажимаешь → n8n забирает пост.

**Логика (как у Сухова):**

1. n8n парсит твой пост из Telegram
2. Агент переписывает в формат Threads (300-500 символов, хук в начале)
3. Генерирует thread_2: "Подробнее разобрал в канале: [ссылка]"
4. Складывает в Airtable → публикация

**Агент:** Отдельный промпт — "Telegram-to-Thread adapter". Правила:

- Сохранить суть, убрать длину
- Первая строка = хук (из поста вытащить самое цепляющее)
- thread_2 = мостик с прямой ссылкой на пост в Telegram

**Частота:** По мере публикаций в TG. Ты контролируешь какие посты ретранслировать.

---

## ПОТОК 3: "ТРЕНДЫ И НОВОСТИ" (автоматический, 1-2 поста/день)

**Источник:** RSS-ленты AI/tech новостей + парсинг тредов конкурентов.

**Триггер:** По расписанию (утро + вечер).

**Логика:**

1. n8n проверяет RSS (TechCrunch AI, Хабр AI, OpenAI blog, Anthropic blog, Product Hunt)
2. Фильтрует: только релевантные AI/автоматизация/контент-маркетинг
3. Агент берёт заголовок + краткое описание новости → пишет Thread-пост
4. Формат: личный комментарий к новости (как soratnik)

**Агент:**

- Вход: заголовок новости + summary (100-200 слов)
- Выход: Thread в стиле "Вот что произошло + почему это важно для вашего бизнеса"
- Тон: как soratnik — живой комментарий, не сухой пересказ
- Без мостика в Telegram (чистая ценность для охвата)

**Частота:** 1-2 поста/день автоматически.

---

## ПОТОК 4: "ФОРМАТЫ КОНКУРЕНТОВ → СВОЙ КОНТЕНТ" (автоматический, 1-2 поста/день)

**Важно:** Мы НЕ переписываем чужие тредсы. Мы анализируем ФОРМАТЫ хуков → генерируем СВОИ тредсы с ПОЛНЫМИ ветками.

**Почему не переписывание:**

1. Парсер видит только Thread 1 — суть (промпты, шаги) в подтредах 2-5 которые невидимы
2. Личные тредсы (корова, хомяк) нельзя адаптировать — это будет фейк
3. Формат важнее содержания — "ЗАПРЕЩАЮ + тема" набирает лайки при ЛЮБОЙ теме

**Источник:** Парсинг тредов конкурентов (уже работает).

**Триггер:** По расписанию или при появлении нового треда с 100+ лайками.

**Логика:**

1. Парсер собирает тредсы с 100+ лайков
2. Code нода фильтрует:
    - Содержит AI/контент/автоматизация/n8n/блог/система/нейросеть → relevant
    - Содержит личные истории не в нише → skip
    - Если непонятно → skip (лучше пропустить чем опубликовать мусор)
3. Агент получает ФОРМАТ хука (не текст целиком) + тему из контент-завода Павла
4. Генерирует ПОЛНЫЙ тред: Thread 1 (хук) + Thread 2-3 (содержание) + Thread финал (мостик в TG)
5. Складывает в Airtable → публикация

**Агент:**

- Вход: format_pattern (тип хука конкурента) + тема из контент-завода
- Выход: thread_1 + thread_2 + thread_3 (полная ветка 3-4 части)
- Thread 1: хук в проверенном формате (300-500 символов)
- Thread 2-3: содержание — шаги, наблюдения, инсайты (200-400 символов каждый)
- Thread финал: мостик в Telegram
- НЕ копировать текст конкурента — только формат

**Библиотека ротации:** Дополнительно создаём 15-20 "вечнозелёных" тредов в нише Павла. Ротируем их каждые 3-5 дней с минимальными вариациями. Петкова доказала: один и тот же пост набирает 100-600 лайков при каждом повторении. Это не спам — это как работает алгоритм Threads.

**Частота:** 1-2 новых поста/день + 1 ротация из библиотеки/день.

## ПОТОК 5: "НАБЛЮДЕНИЯ И ЗАКУЛИСЬЕ" (полуавтоматический, 2-3 раза/неделя)

**Источник:** Ручной триггер. Ты надиктовываешь голосовое или пишешь 2-3 предложения.

**Триггер:** Голосовое в Telegram-бот или текст в Airtable.

**Логика:**

1. Ты отправляешь мысль/наблюдение (голосовое или текст)
2. n8n транскрибирует (если голосовое)
3. Агент разворачивает в Thread-пост
4. Формат: личное наблюдение из практики (как soratnik, но про AI-контент)

**Агент:**

- Вход: короткая мысль (50-200 символов)
- Выход: развёрнутый Thread 300-500 символов с хуком
- Тон: личный, живой, от первого лица
- Может добавлять "А у вас как с этим?" для вовлечения

**Частота:** 2-3 раза в неделю. Единственный поток с твоим участием.

## КРОСС-ПОСТИНГ: THREADS → TWITTER → LINKEDIN

Один пост из Airtable → публикуется на три платформы.

**Адаптация:**

- Threads: до 500 символов, разговорный тон, ветки
- Twitter/X: до 280 символов → агент сокращает Thread до Twitter-формата
- LinkedIn: до 3000 символов → агент разворачивает Thread в более деловой тон

**Реализация:** Code нода после записи в Airtable → три HTTP-запроса к API каждой платформы. Или: три отдельные ноды адаптации (Thread → Twitter shortener, Thread → LinkedIn expander).

**Минимальный вариант:** Публиковать один и тот же текст на все три. Twitter обрежет до 280 — но для старта сойдёт. Потом докрутить адаптацию.

## AIRTABLE СТРУКТУРА

Таблица: **Threads_Queue**

|Поле|Тип|Описание|
|---|---|---|
|id|Auto|—|
|text|Long text|Основной пост|
|thread_2|Long text|Второй пост в ветке (пусто если нет)|
|source_type|Single select|article / telegram / news / competitor / personal|
|format_used|Single select|A / B / C / D / E / F|
|platform|Multi select|threads / twitter / linkedin|
|status|Single select|draft / ready / published / failed|
|scheduled_time|DateTime|Когда опубликовать|
|published_url|URL|Ссылка после публикации|
|source_ref|URL|Откуда взято (статья / TG пост / конкурент)|
|likes|Number|Для трекинга (заполняется парсером позже)|
|created_at|DateTime|Когда создано|
