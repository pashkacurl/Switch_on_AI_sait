# Repo audit — Switch_on_AI_sait

_Last updated: 2026-03-29_

## 1. Executive summary

Репозиторий уже содержит **достаточно материалов для запуска первой рабочей версии сайта**, но они лежат в нескольких слоях зрелости:

1. **Новый основной контур** — `docs/` + `wordpress-theme/`
   - это самая целостная и актуальная система;
   - здесь уже есть стратегия, sitemap, дизайн-принципы, технический план, decision log и рабочая WordPress-тема.

2. **Старый контур / историческое исследование** — `Документация/`
   - полезен как архив решений и источников;
   - часть выводов уже перенесена в `docs/`, часть устарела или конфликтует с новым контуром.

3. **Сырьё бренда, позиционирования и контента** — `iSite/` + `ВКЛЮЧИ ИИ/`
   - это главная база для смыслов, офферов, бренд-системы, контентных рубрик и описания продуктов;
   - материалы сильные, но местами противоречат текущей зрелости проекта: где-то уже описана большая экосистема, хотя по факту часть направлений ещё roadmap.

4. **Текущий/действующий сайт** — `Действующий сайт/`
   - можно использовать как источник текущего живого контента, блоков, CSS-паттернов и медиа;
   - как архитектурную основу использовать не стоит: код тяжёлый, исторический и менее системный, чем новая тема.

### Главный вывод
Для дальнейшей работы логично считать **каноническим источником**:
- **структура/решения** → `docs/`
- **реализация** → `wordpress-theme/`
- **смыслы/продукты/бренд/контент** → `iSite/` + `ВКЛЮЧИ ИИ/`
- **архив и справочный контекст** → `Документация/`, `Действующий сайт/`

## 4. Какие решения уже приняты

## 4.1. По дизайну
Судя по `docs/05-design-principles.md`, `docs/design/`, брендбуку и теме, уже приняты такие решения:
- сайт должен быть **современным, технологичным, но понятным**;
- визуал строится на связке:
  - тёмная база / светлые секции;
  - зелёно-синие AI-акценты;
  - чистый, не перегруженный UX;
- роль дизайна — не “вау ради вау”, а **доверие + ясность + энергия**;
- допускаются технологичные и эффектные элементы, но не в ущерб читабельности и скорости.

## 4.2. По структуре
Уже приняты решения:
- нужен структурированный marketing site, а не просто визитка;
- сайт должен вести пользователя по воронке:
  - понять, что делает Павел / ВКЛЮЧИ ИИ;
  - увидеть оффер;
  - выбрать формат взаимодействия;
  - оставить заявку / перейти в контент / изучить кейсы;
- главные сущности бренда и сайта уже зафиксированы:
  - услуги / внедрение
  - academy
  - club
  - lab
  - tech / B2B
  - блог / контент
  - about / founder

## 4.3. По техстеку
Уже есть решения, что:
- база — **WordPress**;
- есть кастомная тема, которую можно доводить, а не начинать с нуля;
- развитие идёт через WordPress architecture, а не через “полностью всё переделать с нуля на другой стек”;
- блоговая часть и контент-хаб уже продуманы в новой теме.

## 5. Что устарело, конфликтует или требует уточнения

## 5.1. Главный конфликт: зрелость проекта vs зрелость упаковки
Самое заметное противоречие:
- материалы `PDF 1` и часть распаковок говорят: проект ещё на стадии сборки, не всё подтверждено, не все направления живые;
- материалы `PDF 2`, брендбук и местами тема уже описывают **зрелую экосистему**.

### Практический вывод
На сайте нужно разделить:
- **уже есть сейчас**
- **в разработке / скоро запуск**
- **стратегическое направление / roadmap**

Без этого сайт рискует обещать больше, чем реально готово.

## 5.2. Старый WordPress-контур vs новый docs-контур
- `Документация/03_WordPress_техническая_реализация.md` и related historical docs уже не должны считаться главными;
- главный WordPress-контур теперь в `docs/wordpress/` и `wordpress-theme/`.

## 5.3. Личный voice vs corporate voice
- в личных материалах Павел живой, разговорный, местами резкий и неформальный;
- в брендбуке тон чище, более корпоративный, уверенный и “собранный”.

### Практический вывод
Нужны 2 уровня голоса:
- **site brand voice** — чище и собраннее;
- **личные блоки / блог / founder quote** — живее и теплее.

## 5.4. 3D / анимация / wow-элементы
- старые документы подталкивают к более тяжёлому wow-подходу;
- новый контур выглядит более прагматичным.

### Практический вывод
Эффекты — только как акценты. Не делать их обязательной опорой MVP.

- стратегия — есть;

  - устаревшее,
  - архивное,
  - преждевременно масштабное.
