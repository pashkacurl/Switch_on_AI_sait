# Documentation Structure

- Purpose: Описать логику новой doc-системы.
- Status: active
- Last updated: 2026-03-21
- Source of truth: yes
- Related: `docs/INDEX.md`, `docs/archive/README.md`

## Принцип

`docs/` — это не склад заметок, а операционная система проекта. Здесь хранятся:

- актуальная стратегия;
- актуальная архитектура;
- решения;
- backlog;
- execution status;
- WordPress и VPS правила.

## Логика слоев

| Layer | Description |
|---|---|
| `INDEX.md` | Главная карта документации |
| `START-HERE.md` | Быстрый онбординг |
| `00-05` | Аудит, стратегия, структура сайта, контент и дизайн |
| `06-08` | Техплан, SEO/CRO, roadmap запуска |
| `09-14` | Вопросы, decision log, backlog, plans, execution status |
| `wordpress/` | Специализированный слой по WordPress и VPS |
| `archive/` | Явно архивные и reference-only материалы |

## Тематические папки

- Папки `project/`, `research/`, `strategy/`, `sitemap/`, `content/`, `design/`, `engineering/`, `analytics/`, `launch/`, `decisions/`, `backlog/` нужны как устойчивые контейнеры под дальнейшие документы по темам.
- Если папка еще не наполнена рабочими файлами, в ней должен лежать короткий `README.md`, чтобы структура была видна в Git и не превращалась в "пустой намек".
- Как только внутри папки появляется документ, который становится рабочим source of truth, он должен быть добавлен в `docs/INDEX.md`.

## Статусы документов

- `active` — текущий рабочий документ.
- `approved` — принят как базовый reference.
- `candidate` — рабочая гипотеза или вариант.
- `deprecated` — больше не должен вести решения.
- `archived` — только для истории.

## Правило source of truth

- Один слой знаний = один SoT.
- Если появляется новый SoT, старый документ не оставляем “как будто тоже актуален”.
- Любой конфликт между legacy и новым docs решается в пользу нового docs, если это зафиксировано в `docs/10-decision-log.md`.

## Что не считается частью новой doc-системы

- Спонтанные заметки в корне репозитория.
- Старые planning files без статуса.
- Визуальные explorations без выбранного статуса.
