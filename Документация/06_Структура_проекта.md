# Структура проекта и правила ведения

Дата фиксации: 20 марта 2026  
Статус: `active`

## 1) Рабочая модель
Чтобы не ломать действующий сайт, новый цикл делаем в отдельной зоне: `site-v2`.

## 2) Целевая структура папок
```text
Switch_on_AI_sait/
├── Документация/
│   ├── 00_README.md
│   ├── 01_Исследование_дизайна_сайтов.md
│   ├── 02_3D_визуалы_и_анимации.md
│   ├── 03_WordPress_техническая_реализация.md
│   ├── 04_Этап_проектирования.md
│   ├── 05_Рабочий_документ_проекта.md
│   └── 06_Структура_проекта.md
├── style-previews/
│   └── (быстрые визуальные прототипы)
└── site-v2/
    ├── index.html
    ├── assets/
    │   ├── css/
    │   ├── js/
    │   └── img/
    ├── components/
    ├── pages/
    ├── data/
    └── docs/
```

## 3) Назначение папок `site-v2`
1. `assets/css` — стили проекта.
2. `assets/js` — сценарии интерфейса.
3. `assets/img` — изображения, иконки, графика.
4. `components` — переиспользуемые блоки (hero, cards, faq, cta).
5. `pages` — страницы сайта (главная, услуги, кейсы, блог).
6. `data` — JSON/данные для прототипов или моков.
7. `docs` — локальные технические заметки по реализации.

## 4) Правила именования
1. Файлы: `kebab-case` (пример: `main-hero.css`, `cta-block.js`).
2. Компоненты: префикс по блоку (`hero-*`, `faq-*`, `case-*`).
3. Версии прототипов: `style-0X-*` в папке `style-previews`.

## 5) Правила работы по этапам
1. Проектирование: только структура, контентный каркас, UI-система.
2. Реализация: перенос утверждённой структуры в код.
3. Функционал: отдельный backlog и отдельное ТЗ.
4. Любые незапланированные функции не идут сразу в код, а фиксируются в документе.

## 6) Definition of Done для задачи
1. Есть понятный результат в интерфейсе.
2. Есть проверка на mobile/tablet/desktop.
3. Нет конфликтов со структурой проекта.
4. Изменение отражено в документации при необходимости.
