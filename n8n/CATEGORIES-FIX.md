# Фикс: категории в ноде "Создание поста"

## Проблема

```
400 - categories[0] не принадлежит к типу integer
```

WordPress REST API требует `categories` как массив чисел `[5]`, а нода передаёт строку `["5"]`.

## Причина

В ноде "Создание поста" параметр `categories` установлен как:
```
={{ $('Переменные').item.json.wp_category_id }}
```
Airtable хранит значение как текст → n8n передаёт строку → WP отклоняет.

## Решение

### Вариант 1: parseInt (рекомендуемый)

В ноде **"Создание поста"** — параметр `categories`:

```
={{ [parseInt($('Переменные').item.json.wp_category_id) || 1] }}
```

- `parseInt()` — конвертирует строку "5" в число 5
- `|| 1` — фоллбэк на Uncategorized если поле пустое
- `[...]` — оборачивает в массив как ожидает WP

### Вариант 2: захардкодить временно

Если все статьи идут в одну категорию:

```
={{ [5] }}
```

Замени `5` на реальный ID рубрики из WordPress.

### Вариант 3: убрать параметр

Если категории пока не настроены — просто удали параметр `categories` из ноды. Посты будут падать в дефолтную рубрику.
